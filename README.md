# LAN голосовые звонки (2 ПК в одной локальной сети)

Приложение для голосовых звонков между двумя компьютерами в одной LAN-сети.
Интерфейс построен на PySide6, сигнализация и передача аудио работают по UDP.

## Что это за приложение
- Звонки между двумя ПК в локальной сети без внешних серверов.
- Графический интерфейс с настройками сети, устройств и аудиообработки.
- Журнал событий и отображение текущего состояния звонка.
- Поддержка шумоподавления через опциональный RNNoise-бэкенд.

## Установка
```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
```

### Опционально: шумоподавление RNNoise
Можно установить один из поддерживаемых биндингов:
```bash
pip install rnnoise
# или
pip install pyrnnoise
```

Если биндинг не установлен, приложение продолжит работать, но шумоподавление будет недоступно.

## Запуск
```bash
python -m app.main
```

## Как узнать IP
### Windows
```bat
ipconfig
```
Используйте IPv4-адрес активного сетевого адаптера.

### Linux/macOS
```bash
ip a
# или
ifconfig
```
Используйте IP интерфейса, который находится в вашей локальной сети.

## Как звонить между двумя ПК
1. Запустите приложение на обоих ПК.
2. В блоке **Локальные настройки** на каждом ПК задайте порты и нажмите **Применить / Перезапустить прослушивание**.
3. В блоке **Аудиоустройства** выберите микрофон и устройство вывода (или нажмите **Обновить устройства**).
4. На первом ПК в **Настройки второго ПК** укажите IP и порты второго ПК.
5. Нажмите **Позвонить**.
6. На втором ПК нажмите **Принять** или **Отклонить**.
7. Для завершения вызова нажмите **Завершить**.

## Что означают локальные настройки и настройки второго ПК
- **Локальные настройки** — параметры текущего ПК:
  - локальный порт signaling;
  - локальный аудиопорт;
  - локальный IP.
- **Настройки второго ПК** — куда отправлять вызов:
  - IP второго компьютера;
  - порт signaling второго ПК;
  - аудиопорт второго ПК.

## Как работают аудиопараметры
### Чувствительность микрофона
Изменяет усиление микрофонного сигнала перед отправкой в сеть:
- больше 1.0 — усиливает;
- меньше 1.0 — ослабляет.

### Шумовой порог и порог шума
- Если **Шумовой порог включен**, приложение оценивает уровень сигнала (RMS).
- Если уровень ниже **Порога шума**, текущий фрейм заменяется тишиной.

### Шумоподавление
- При доступном RNNoise-бэкенде и включенной опции сигнал проходит через шумоподавление.
- Если бэкенд недоступен, приложение работает без шумоподавления.

### Выключить микрофон (Mute)
- Индикатор входного уровня продолжает работать.
- В сеть отправляется тишина вместо живого микрофона.

## Troubleshooting
Если соединение есть, но звука нет:
- Проверьте выбранные устройства ввода/вывода в GUI.
- Проверьте, что микрофон и динамики не заняты другим приложением.
- Убедитесь, что на обоих ПК правильно указаны IP и порты.
- Разрешите Python в брандмауэре (Windows Firewall) или откройте нужные UDP-порты.
- Проверьте, что оба ПК находятся в одной LAN и пингуются.

## Текущие ограничения
- Используется raw PCM (без Opus-кодека).
- Нет NAT traversal и интернет-звонков.
- Нет установщика.
- Нет эхоподавления.
