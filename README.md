# LAN голосовые звонки (2 ПК в одной локальной сети)

Приложение для голосовых звонков между двумя компьютерами в одной LAN-сети.
Интерфейс построен на PySide6, сигнализация и передача аудио работают по UDP.

## Что это за приложение
- Звонки между двумя ПК в локальной сети без внешних серверов.
- Графический интерфейс с настройками сети, устройств и аудиообработки.
- Журнал событий и отображение текущего состояния звонка.
- Поддержка шумоподавления через опциональный RNNoise-бэкенд.

## Установка
```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
```

### Опционально: шумоподавление RNNoise
Можно установить один из поддерживаемых биндингов:
```bash
pip install rnnoise
# или
pip install pyrnnoise
```

Если биндинг не установлен, приложение продолжит работать, но шумоподавление будет недоступно.

## Запуск
```bash
python -m app.main
```

## Сохранение и автозагрузка настроек
Приложение сохраняет пользовательские параметры между запусками через `QSettings` (без базы данных):
- local signaling port;
- local audio port;
- peer IP;
- peer signaling port;
- peer audio port;
- selected input device;
- selected output device;
- mic gain;
- noise gate enabled;
- noise gate threshold;
- noise suppression enabled;
- mute microphone.

### Где хранятся настройки
`QSettings` использует стандартную пользовательскую папку ОС:
- **Windows:** реестр пользователя (ветка организации/приложения);
- **Linux:** обычно файл в `~/.config`;
- **macOS:** стандартное хранилище Preferences.

Организация и имя приложения задаются константами `APP_ORG` и `APP_NAME`.

### Как работает автозагрузка
При старте:
- значения загружаются и подставляются в UI;
- порты валидируются (если некорректны — берутся default и это пишется в лог);
- если сохранённое аудиоустройство отсутствует — используется default, в лог пишется fallback.

## Проверки перед звонком
Перед нажатием **Позвонить** выполняются проверки:
- указан peer IP;
- signaling listener запущен;
- есть устройство вывода;
- есть устройство ввода, если микрофон не muted.

Если проверка не проходит — показывается понятное сообщение на русском и запись в журнал.

## Диагностический блок
В интерфейсе добавлен компактный read-only блок:
- версия приложения;
- статус listener;
- статус шумоподавления;
- активные устройства (ввод/вывод).

## Иконка приложения
Подготовлена папка `assets/`.

Чтобы подключить иконку:
1. Положите файл `icon.ico` в `assets/icon.ico`.
2. Запустите приложение или сборку — иконка подхватится автоматически.

Если файла нет, приложение работает без падения.

## Сборка Windows .exe через PyInstaller
Добавлен скрипт `build_windows.bat`.

### 1) Установить PyInstaller (отдельно)
PyInstaller не добавлен в runtime-зависимости приложения, установите его отдельно:
```bash
pip install pyinstaller
```

### 2) Запустить сборку
На Windows в корне проекта:
```bat
build_windows.bat
```

Скрипт собирает GUI-приложение:
- без консоли (`--noconsole`);
- с entrypoint `app/main.py`;
- с optional `assets/icon.ico`, если файл существует.

## Если после сборки не находятся аудиоустройства
- Запустите `.exe` от имени пользователя с доступом к аудио устройствам.
- Проверьте, что устройства видны в системе и не заняты другим приложением.
- Нажмите **Обновить устройства** в GUI.
- Если сохранённые устройства недоступны, приложение автоматически переключится на default и напишет это в лог.

## Как узнать IP
### Windows
```bat
ipconfig
```
Используйте IPv4-адрес активного сетевого адаптера.

### Linux/macOS
```bash
ip a
# или
ifconfig
```
Используйте IP интерфейса, который находится в вашей локальной сети.

## Как звонить между двумя ПК
1. Запустите приложение на обоих ПК.
2. В блоке **Локальные настройки** на каждом ПК задайте порты и нажмите **Применить / Перезапустить прослушивание**.
3. В блоке **Аудиоустройства** выберите микрофон и устройство вывода (или нажмите **Обновить устройства**).
4. На первом ПК в **Настройки второго ПК** укажите IP и порты второго ПК.
5. Нажмите **Позвонить**.
6. На втором ПК нажмите **Принять** или **Отклонить**.
7. Для завершения вызова нажмите **Завершить**.

## Текущие ограничения
- Используется raw PCM (без Opus-кодека).
- Нет NAT traversal и интернет-звонков.
- Нет установщика.
- Нет эхоподавления.
